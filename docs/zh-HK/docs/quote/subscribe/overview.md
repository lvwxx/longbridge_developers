Streaming 透過 WebSocket 訂閱的方式提供即時行情數據。無需輪詢，只需訂閱指定標的和數據類型，服務端會在行情變動時主動推送更新到你的連線。

## 運作原理

1. **建立連線** — 透過 SDK 或[二進位通訊協定](../../socket/protocol/overview)連線到 Longbridge WebSocket 服務。
2. **訂閱** — 呼叫[訂閱](./subscribe)介面，指定需要接收推送的標的和數據類型（報價、盤口、逐筆、券商佇列）。
3. **接收推送** — 行情發生變動時，服務端即時推送數據到你的連線。
4. **取消訂閱** — 呼叫[取消訂閱](./unsubscribe)停止接收特定標的或數據類型的推送。

## 訂閱類型

| 類型 | 說明 |
|------|------|
| `Quote` | 即時買賣價及成交價 |
| `Depth` | Level 2 買賣盤口 |
| `Brokers` | 券商佇列數據 |
| `Trade` | 即時逐筆成交 |

## 訂閱 vs. 直接查詢

訂閱適合需要即時增量更新的場景。如需一次性或按需查詢，可使用以下拉取介面：

| 數據 | 拉取介面 |
|------|----------|
| 報價 | [實時報價](../stocks/quote) |
| 盤口 | [盤口](../stocks/depth) |
| 券商佇列 | [券商佇列](../stocks/brokers) |
| 逐筆明細 | [逐筆明細](../stocks/trade) |
| 分時數據 | [分時數據](../stocks/intraday) |
| K 線 | [K 線](../stocks/candlestick) |

## 注意事項

- 訂閱狀態與當前連線綁定，斷線後自動清除。
- 可透過[獲取訂閱資訊](./subsciption)隨時查詢當前活躍的訂閱列表。
- 可訂閱的數據類型取決於你的行情權限，詳見[行情權限等級](../overview#行情權限等級)。
